# Discord MCP 선택 전략 분석

여러 MCP 서버가 연결된 환경에서 Discord 사용자가 특정 MCP를 선택하는 방안을 분석한다.

## 방안 비교

| 방안 | UX | 난이도 | 모바일 | 추천 |
|------|:--:|:------:|:-----:|:----:|
| 슬래시 커맨드 | 5 | 중간 | 좋음 | **1순위** |
| 채널별 지정 | 4 | 낮음 | 좋음 | **2순위** |
| 이모지 반응 | 3 | 중간 | 좋음 | 3순위 |
| 버튼 인터랙션 | 4 | 중간 | 좋음 | 3순위 |
| @멘션 | 3 | 낮음 | 보통 | 현재 |
| 키워드 감지 | 3 | 중간 | 좋음 | 5순위 |

## 1. 슬래시 커맨드

```
/gcp status    → GCP MCP
/oci list       → OCI MCP
/db query       → DB MCP
```

- 공식 Discord 기능
- 자동완성 지원
- 파라미터 검증

## 2. 채널별 지정

| 채널 | 기본 MCP |
|------|----------|
| #gcp-모니터링 | gcp-mcp |
| #oci-모니터링 | oci-mcp |
| #일반 | broadcast |

- 채널만 선택하면 됨
- 자연스러운 대화

## 3. 이모지 반응

```
사용자: 서버 상태 확인
봇: 🔵 GCP  🟠 OCI  🟢 DB

사용자가 🔵 클릭 → GCP MCP 응답
```

- 직관적
- 모바일 친화적

## 4. 키워드 감지

| 키워드 | MCP |
|--------|-----|
| gcp, 구글 | gcp-mcp |
| oci, 오라클 | oci-mcp |
| db, 데이터베이스 | db-mcp |

- 자동 감지
- 별도 액션 불필요

## 추천: 하이브리드 4단계

| 순위 | 방식 | 예시 |
|:----:|------|------|
| 1 | /커맨드 | /gcp status |
| 2 | @멘션 | @gcp-monitor |
| 3 | 키워드 | gcp 서버 |
| 4 | 채널별 | #gcp-모니터링 |

## Fallback 순서

```
1. 슬래시? → 해당 MCP
2. @멘션? → 해당 MCP
3. 키워드? → 해당 MCP
4. 채널 기본? → 해당 MCP
5. 없음 → Broadcast
```

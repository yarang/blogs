## GigaToken: 1000배 더 빠른 토크나이저로의 여정

최근 LLM(대규모 언어 모델)을 활용한 서비스를 개발할 때, 단순히 추론(Inference) 속도만큼이나 중요한 것이 바로 **전처리 속도**입니다. Hacker News에서 "GigaToken: ~1000x faster Language model tokenization"이라는 흥미로운 주제를 보았습니다.

우리가 개발 중인 에이전트 런타임인 **ZeroClaw** 역시 대규모 텍스트 처리를 자주 수행합니다. 오늘은 GigaToken의 아이디어를 참고하여, Rust로 어떻게 고성능 토크나이저를 구현할 수 있는지, 그리고 실제 코드로 어떻게 최적화할 수 있는지 살펴보겠습니다.

### 왜 토크나이저인가?

LLM 서비스를 운영하다 보면 토큰화 과정에서 병목이 발생하는 경우가 많습니다. 특히 Python 기반의 Hugging Face `tokenizers` 라이브러리는 편리하지만, Rust로 작성된 코어임에도 불구하고 Python 바인딩 오버헤드와 GIL(Global Interpreter Lock) 때문에 멀티스레딩 환경에서 한계가 있습니다.

GigaToken의 핵심은 **SIMD(Single Instruction, Multiple Data)** 명령어와 **효율적인 메모리 관리**를 통해 이를 극복하는 데 있습니다.

### Rust로 고성능 토크나이저 질겅보기

기존 방식은 문자열을 순차적으로 스캔하며 정규식을 매칭합니다. Rust에서 이를 더 빠르게 만들기 위해서는 `regex` 크레이트의 최적화 기능을 활용하거나, 바이트 단위의 처리를 직접 구현하는 방법이 있습니다.

아래는 간단한 BPE(Byte Pair Encoding) 스타일의 토크나이저를 Rust로 구현하고, 이를 병렬 처리하는 예제입니다.

#### 1. 기본 BPE 구조 및 병렬 처리 (Rayon 활용)

가장 큰 병목은 긴 텍스트를 쪼개는 과정입니다. Rust의 병렬 처리 라이브러리인 `rayon`을 사용하여 입력 텍스트를 청크 단위로 나누어 처리할 수 있습니다.

```rust
use rayon::prelude::*;
use std::collections::HashMap;

struct Tokenizer {
    vocab: HashMap<String, u32>,
}

impl Tokenizer {
    fn new() -> Self {
        // 예시를 위한 간단한 어휘집 초기화
        let mut vocab = HashMap::new();
        vocab.insert("hello".to_string(), 1);
        vocab.insert("world".to_string(), 2);
        vocab.insert(" ".to_string(), 3);
        Tokenizer { vocab }
    }

    // 단순화된 토큰화 함수 (실제 BPE 로직은 더 복잡함)
    fn tokenize_chunk(&self, chunk: &str) -> Vec<u32> {
        chunk.split_whitespace()
            .filter_map(|word| self.vocab.get(word).copied())
            .collect()
    }

    // 병렬 처리를 통한 토큰화
    fn tokenize_parallel(&self, text: &str) -> Vec<u32> {
        // 텍스트를 라인 단위로 나누어 병렬 처리
        let chunks: Vec<&str> = text.lines().collect();
        
        chunks.par_iter() // 병렬 이터레이터
            .flat_map(|chunk| self.tokenize_chunk(chunk))
            .collect()
    }
}

fn main() {
    let tokenizer = Tokenizer::new();
    let text = "hello world\nhello world\nhello world"; // 반복되는 텍스트
    
    let tokens = tokenizer.tokenize_parallel(text);
    println!("Tokens: {:?}", tokens);
}
```

#### 2. SIMD를 활용한 문자열 검색 (SIMD 최적화)

GigaToken에서 언급하는 핵심 최적화 기법은 **SIMD**를 활용해 여러 문자를 동시에 비교하는 것입니다. Rust에서는 `std::simd` 모듈(아직 nightly이거나 특정 크레이트 필요)을 통해 이를 구현할 수 있습니다.

다음은 `memchr` 라이브러리가 내부적으로 수행하는 것과 유사하게, 구분자(Whitespace)를 찾는 과정을 시뮬레이션한 코드입니다. 실제 프로덕션 코드에서는 `memchr`나 `simd-json` 같은 잘 최적화된 라이브러리를 사용하는 것이 좋습니다.

```rust
// SIMD 개념을 활용한 whitespace 카운팅 (가상의 예시)
// 실제로는 target_feature 속성을 사용해야 합니다.

#[cfg(target_arch = "x86_64")]
use std::arch::x86_64::*;

/// 단순히 공백을 찾는 함수이나, SIMD를 사용하면 16/32바이트 단위로 검색 가능
/// 이 코드는 개념 설명을 위한 것입니다.
fn count_spaces_simd(text: &[u8]) -> usize {
    let mut count = 0;
    let len = text.len();
    let chunks = len / 32; // AVX2는 256비트(32바이트)
    
    // 기본 루프
    for i in 0..chunks {
        // let vec = _mm256_loadu_si256(...);
        // SIMD 연산을 통해 공백 마스크 생성 및 카운트
        // (실제 구현은 unsafe 블록과 어셈블리 지식이 필요함)
    }
    
    // 나머지 부분 처리
    for i in (chunks * 32)..len {
        if text[i] == b' ' {
            count += 1;
        }
    }
    count
}
```

### ZeroClaw 적용 계획

우리는 **ZeroClaw** 프로젝트의 멀티 에이전트 통신 부분에 이러한 고성능 토크나이저를 도입할 계획입니다. 기존에는 Python 기반의 서비스를 호출했으나, 이를 Rust 네이티브 모듈로 변경하여 다음과 같은 이득을 기대합니다.

1.  **지연 시간 감소**: Python 오버헤드 제거 및 SIMD 가속을 통해 토큰화 속도 10~100배 개선
2.  **메모리 사용량 최적화**: 불필요한 문자열 복사 제거 및 `bytes` 처리
3.  **동시성 처리 강화**: `rayon`을 통한 멀티코어 활용 극대화

### 결론

"GigaToken" 게시물이 보여주듯, 토크나이저는 LLM 파이프라인에서 단순한 유틸리티가 아니라 성능의 핵심 요소입니다. Rust의 안전성과 제로 비용 추상화, 그리고 강력한 병렬 처리 라이브러리는 이를 구현하기에 최적의 환경을 제공합니다.

Python에서 `pip install`로 해결하던 시대는 저물어가고, Rust로 병목을 제거하는 시대가 오고 있습니다. ZeroClaw의 다음 버전에서 이러한 최적화가 적용된 모듈을 만나보시길 바랍니다.

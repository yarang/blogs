# Detecting LLM-Generated Text: Catching 'Deepfake' Text with Classical Machine Learning

Recent advancements in AI models (like Kimi, LM Studio, etc.) have brought the quality of LLM-generated text to a level almost indistinguishable from human writing. As AI tools, such as NotebookLM's integration into Gemini Notebook, become deeply ingrained in our daily writing, the question "Is this text written by AI?" has moved beyond mere curiosity to become a critical issue for security and authenticity.

A recent article on Hacker News, 'Detecting LLM-Generated Texts with Classical Machine Learning,' demonstrated that LLM-generated text can be effectively detected using traditional machine learning (Classical ML) techniques, without the need for large models. In this post, we will introduce how to implement this method directly and build a lightweight, efficient detection system.

## Why 'Classical' Machine Learning?

Using large models like GPT-4 for LLM detection can be like "using a sledgehammer to crack a nut." The high cost and latency make it impractical for real-time monitoring systems (e.g., log analysis in [Cloud Monitor] or [blog-api-server]).

On the other hand, traditional algorithms like Random Forest or SVM offer the following advantages:

1.  **Low Cost**: No GPU required; fast inference even on a single CPU core.
2.  **Interpretability**: Easy to explain why the model made a certain judgment.
3.  **Ease of Deployment**: Small model size allows for easy integration into edge devices or legacy servers.

The core of this approach lies in capturing statistical patterns inherent in LLMs, such as **'Perplexity'** and **'Burstiness'**.

## Detection System Architecture

Let's design a simple pipeline for practical application. This structure can also be utilized as a guardrail component operating within the [ZeroClaw] agent runtime.

1.  **Data Preprocessing**: Tokenize text and extract statistical features like n-grams.
2.  **Feature Extraction**: Calculate quantitative metrics such as word frequency, sentence length, and vocabulary diversity.
3.  **Classification**: Determine "AI-Generated" or "Human Written" using a trained Random Forest model.

## Practical Code Example (Python)

We will build a simple detection model using Scikit-learn. While a larger dataset is necessary for a real production environment, here we will focus on understanding the core logic.

### 1. Data Preparation and Feature Extraction

LLM text typically has lower **'Perplexity'** and more uniform sentence structures than human text. We'll write functions to quantify these.

```python
import re
import numpy as np
from collections import Counter

def extract_features(text):
    # 1. Average Sentence Length
    sentences = re.split(r'[.!?]', text)
    avg_sentence_length = np.mean([len(s.split()) for s in sentences if s.strip()])

    # 2. Vocabulary Richness (Unique words / Total words)
    words = re.findall(r'\w+', text.lower())
    vocab_richness = len(set(words)) / len(words) if words else 0

    # 3. Punctuation Frequency (LLMs often have different punctuation patterns)
    punct_count = len(re.findall(r'[,;:]', text))
    
    # 4. Check for Unnecessary Repetition (Simplified Burstiness measure)
    word_counts = Counter(words)
    most_common_count = word_counts.most_common(1)[0][1] if word_counts else 0
    repetition_ratio = most_common_count / len(words) if words else 0

    return [avg_sentence_length, vocab_richness, punct_count, repetition_ratio]
```

### 2. Model Training and Prediction

Assuming we have a pre-collected dataset, we will train a `RandomForestClassifier`.

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split

# Hypothetical training data (sufficient dataset needed in reality)
# Features: [Average Sentence Length, Vocabulary Richness, Punctuation Count, Repetition Ratio]
X = np.array([
    [15.0, 0.6, 5, 0.05], # Human-written characteristics (hypothetical)
    [18.5, 0.65, 8, 0.04],
    [12.0, 0.5, 3, 0.02], 
    [25.0, 0.4, 2, 0.01], # AI-written characteristics (hypothetical) - longer sentences, repetitive vocabulary
    [28.0, 0.35, 1, 0.01],
])
y = np.array([0, 0, 0, 1, 1]) # 0: Human, 1: AI

# Train the model
clf = RandomForestClassifier(n_estimators=100, random_state=42)
clf.fit(X, y)

def is_ai_generated(text):
    features = [extract_features(text)]
    prediction = clf.predict(features)
    probability = clf.predict_proba(features)
    
    return "AI Generated" if prediction[0] == 1 else "Human Written", probability[0]
```

### 3. Execution Results

Now we can make a judgment on new text.

```python
# Test text (LLM style)
test_text = """
Artificial intelligence is transforming the world. Technology is advancing rapidly. 
We must consider the implications of progress. The future is bright for innovation.
"""

label, prob = is_ai_generated(test_text)
print(f"Result: {label}")
print(f"Confidence: {prob}")
```

## Limitations and Improvement Measures

This 'classical' approach is not perfect. LLMs can be vulnerable to Adversarial Attacks where they intentionally manipulate 'Perplexity.' Therefore, a hybrid approach is recommended for real-world systems:

1.  **Combine with Log Analysis**: As seen in the [blog-api-server] logging improvement case, analyzing user behavior patterns (Context) along with text characteristics can increase accuracy.
2.  **Utilize MCP Tools**: Integrate with [Discord MCP] or [Cloud Monitor] to build a workflow that sends immediate alerts if suspicious text generation attempts are detected.
3.  **Zero-Shot Classifiers**: A method involves adding a preliminary filtering step that asks the LLM itself, without separate training, to "judge if this text was written by AI."

## Conclusion

By leveraging lightweight and fast 'classical machine learning' without relying solely on massive models, you can build real-time text filtering systems much more efficiently. This is an optimal technology to utilize as a guardrail within automated environments like ZeroClaw or MCP agents.

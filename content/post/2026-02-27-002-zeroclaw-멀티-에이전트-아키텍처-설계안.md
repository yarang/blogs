# ZeroClaw 멀티 에이전트 아키텍처 설계안

## 개요

ZeroClaw의 멀티 에이전트 아키텍처는 현재 2단계 구축 중에 있으며, 단일 에이전트 모델에서 진화하는 고급 협력 시스템을 구현합니다.

### 현재 상태
- **Phase 1**: In-Process Delegation (기능 완료)
- **Phase 2**: File-Based Multi-Agent Architecture (개발 중)

---

## 1. 설계 철학 및 원칙

| 원칙 | 멀티 에이전트 적용 |
|------|-------------------|
| **KISS** | 단순 통신 프로토콜, 요청/응답 패턴 |
| **YAGNI** | 필수적 기능만 구현 |
| **SRP+ISP** | 단일 책임 에이전트 분리 |
| **Fail Fast** | 명시적 오류 처리 |
| **Secure** | 최소 권한 원칙, 에이전트별 도구 접근 제어 |

---

## 2. 아키텍처 구조

### 통신 계층

```
┌─────────────────────────────────────────┐
│           Application Layer              │
│  Research Agent │ Code Agent │ Test Agent│
└─────────────────────────────────────────┘
│           Message Bus Layer              │
│         NATS/Redis Pub/Sub               │
└─────────────────────────────────────────┘
│           Transport Layer                │
│   gRPC │ WebSocket │ Unix Socket         │
└─────────────────────────────────────────┘
```

### 핵심 인터페이스

```rust
trait ResearchAgent {
    fn research(&self, topic: &str) -> Result<String>;
    fn synthesize(&self, sources: Vec<String>) -> Result<String>;
}

trait CodeAgent {
    fn generate_code(&self, spec: &str) -> Result<String>;
    fn refactor_code(&self, code: &str) -> Result<String>;
}

trait TestAgent {
    fn run_tests(&self, code: &str) -> Result<TestResults>;
}
```

## 3. 에이전트 정의 파일 구조

```toml
# agents/researcher.toml
[agent]
id = "researcher"
name = "Research Agent"
version = "1.0.0"

[execution]
mode = "subprocess"
command = "zeroclaw"

[provider]
name = "openrouter"
model = "anthropic/claude-sonnet-4-6"

[[tools]]
name = "web_search"
enabled = true
```

---

## 4. CLI 명령어

```bash
zeroclaw agent list          # 에이전트 목록
zeroclaw agent show <id>     # 에이전트 상세
zeroclaw agent run --agent-id researcher --prompt "연구 주제"
zeroclaw agent validate <id> # 에이전트 검증
```

# 블로그 MCP 시스템 동작 방식 정리

## 개요

이 시스템은 Claude Code에서 MCP(Model Context Protocol)를 통해 블로그 포스트를 관리하는 자동화 시스템입니다.

## 아키텍처

```
┌──────────────────┐     MCP      ┌──────────────────┐     HTTP     ┌──────────────────┐
│   Claude Code    │ ──────────▶ │   MCP Client     │ ──────────▶ │   API Server     │
│   (로컬)         │              │   (로컬)         │              │   (OCI 서버)     │
└──────────────────┘              └──────────────────┘              └──────────────────┘
                                                                            │
                                                                            ▼
                                                                   ┌──────────────────┐
                                                                   │   Git 저장소     │
                                                                   │   (GitHub)       │
                                                                   └──────────────────┘
```

## 구성 요소

### 1. MCP 클라이언트
- **위치**: `blog-api-server/mcp_client/mcp_blog_client.py`
- **역할**: Claude Code와 API 서버 간의 통신 중계
- **프로토콜**: MCP (stdio 기반)

### 2. API 서버
- **위치**: `oci-yarang-ec1:/var/www/blog-api/`
- **URL**: `https://blog.fcoinfup.com/api/`
- **프레임워크**: FastAPI + Uvicorn
- **포트**: 8000

### 3. 로깅 시스템
강화된 로깅 기능이 포함되어 있습니다:
- UUID 기반 요청 추적
- 요청/응답 바디 로깅 (민감 정보 마스킹)
- 타이머존 정보 포함
- 호출자 정보 (파일명, 라인번호)

## 사용 가능한 도구

| 도구 | 설명 |
|------|------|
| `blog_create` | 새 포스트 작성 |
| `blog_list` | 포스트 목록 조회 |
| `blog_get` | 특정 포스트 조회 |
| `blog_update` | 포스트 수정 |
| `blog_delete` | 포스트 삭제 |
| `blog_search` | 포스트 검색 |
| `blog_status` | Git 상태 확인 |
| `blog_sync` | Git 동기화 |

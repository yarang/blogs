# 파일 권한 검증 — S_IRWXG vs S_IWOTH

**발생일**: 2026-05-03
**카테고리**: 보안 + 운영

## 문제

토큰 파일 권한이 640(rw-r-----)인데 Python 코드가 이를 거부.

## 비트 마스크 분석

```
stat.S_IRWXG = 0o070  (그룹: rwx)
stat.S_IRWXO = 0o007  (기타: rwx)
stat.S_IWOTH = 0o002  (기타: -w-)

# 640 = 0o640
0o640 & (0o070 | 0o007) = 0o640 & 0o077 = 0o040  → 거부! (그룹 읽기 비트)
0o640 & 0o002 = 0o000  → 허용 (world-writable 아님)
```

## 판단 기준

| 검사 대상 | 의미 | 적절한가? |
|-----------|------|-----------|
| `S_IRWXG \| S_IRWXO` | 그룹/기타 모든 권한 거부 | 과도함 — 640 정상 파일 거부 |
| `S_IWOTH` | 기타 쓰기만 거부 | 적절 — 실제 위협에 집중 |
| `S_IRWXO` | 기타 모든 권한 거부 | 중간 — 644 읽기 허용도 거부 |

## 결론

보안 검증은 **실제 위협 모델**에 맞춰야 한다.
- world-writable(기타 쓰기)은 누구나 파일을 변조할 수 있으므로 반드시 거부
- 그룹 읽기(640)는 같은 그룹 사용자가 읽을 수 있을 뿐, 보안 위협이 아님
- "너무 엄격한 검증은 너무 느슨한 검증만큼 해롭다"
